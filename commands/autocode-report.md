# /autocode-report — Generate Session Summary

Generate a shareable markdown report of AutoCode's work on this repository.

## Output Format

Start with the branded header:
```
  ┌─────────────────────────────────────┐
  │  AutoCode — Report                  │
  │  <repo name> · v4.1                 │
  └─────────────────────────────────────┘
```

## Steps

### 1. Read Manifest

Read `autocode.manifest.json`. Extract repo name, language, framework, current difficulty level, and coverage targets.

### 2. Read Memory Files

Read all memory files from `.autocode/memory/`:
- `velocity.md` — cycle history
- `coverage.md` — coverage progression
- `fixes.md` — successful changes
- `failures.md` — failed attempts
- `lessons.md` — accumulated learnings
- `costs.md` — cost estimates (if exists)

### 3. Compute Stats

From the memory files, calculate:
- **Total cycles**: Count all cycle entries in velocity.md
- **Success rate**: Successes / total cycles (as percentage)
- **Total tests added**: Sum of "Tests added" from fixes.md
- **Total PRs**: Count of SUCCESS entries with PR URLs
- **Coverage delta**: First recorded overall coverage → latest overall coverage
- **Estimated cost**: Sum from costs.md (if available)
- **Time span**: First cycle timestamp → last cycle timestamp
- **Current level**: From manifest
- **Top lesson**: Most recent success pattern from lessons.md

### 4. List PRs

Extract all PR URLs from velocity.md. For each, use `gh pr view <URL> --json title,state,mergedAt` to get the title and merge status.

### 5. Generate Report

Output the report directly (don't write to a file — the user can copy it):

```markdown
# AutoCode Report — <repo_name>

> Autonomous code improvements by [AutoCode](https://github.com/JustinBeaudry/autocode)

## Summary

| Metric | Value |
|--------|-------|
| Cycles | <total> (<successes> success, <failures> failed) |
| Success Rate | <X>% |
| Tests Added | <total_tests> |
| PRs Shipped | <count> (<merged> merged, <open> open) |
| Coverage | <start>% → <current>% (+<delta>%) |
| Difficulty | Level <N> — <description> |
| Estimated Cost | ~$<amount> |
| Duration | <time_span> |

## PRs

| # | Title | Tests | Status |
|---|-------|-------|--------|
| <number> | <title> | <test_count> | Merged / Open |
| ... | ... | ... | ... |

## Coverage Progression

```
<file1>: 0% → 45% (+45%)
<file2>: 0% → 62% (+62%)
<file3>: 0% → 38% (+38%)
...
```

## Key Lessons Learned

- <lesson 1>
- <lesson 2>
- <lesson 3>

## Failures & Skips

- <file>: <reason> (skipped after 3 failures)
- <file>: <reason> (Reviewer rejected 2x)

---

*Generated by [AutoCode](https://github.com/JustinBeaudry/autocode) — autonomous code factory for Claude Code*
```

### 6. Suggest Sharing

After generating the report, suggest:
- "Copy this report to share in a PR, README, or team update."
- "To save to a file: copy the output above, or run this command again and pipe to a file."

## Notes

- If memory files don't exist or are empty, show "No data yet — run `/autocode` first."
- If `gh` CLI is not available, skip the PR status check and just show PR URLs from memory
- Keep the report concise — it should fit in a single screen or tweet thread
